Release description

Product overview

Fast Models are accurate, flexible programmer's view models of Arm IP, allowing you to develop software such as drivers, firmware, OS and applications prior to silicon availability. They allow full control over the simulation, including profiling, debug and trace. Fast Models can be exported to SystemC and TLM 2.0, allowing integration into the wider SoC design process.

Detailed documentation can be found in the Docs subfolder of your Fast Models Portfolio installation, and on developer.arm.com

Some models are made available to provide early access before they are complete, and may have limitations. The models have one of the following quality categories:

  • Alpha:
    The model implementation is at an early stage and is likely to change in future releases. There might be significant defects or limitations in the model.
  • Preliminary support:
    The model implementation is complete or almost complete. Testing is nearly to the level of a fully-supported model. However, changes might still occur and there are likely to be known defects present.
  • Fully supported:
    Also called Release quality. Support is complete and the model has been tested as fully as possible. The modeled IP has reached its LAC milestone.

Issues listed in this document have identifiers of the form [SDDKW-nnnn]. These are Arm internal references only.

Downloading and installation

You must download the 11.26 deliverables from the Arm delivery server, Product Download Hub (PDH), and then unpack the contents.

Prerequisites

  • You must have the necessary access rights for PDH.
  • Ensure you have sufficient space in your download area. Check the file size stated in PDH.

General installation guidelines

As a general rule, always download installers directly from the Arm website and check the MD5 hashes listed for each download. Arm recommends:

  • Downloading and storing installers into directories that have limited write access to prevent unauthorized tampering prior to executing the installer. Typically, only the user downloading the installer should have write access to the target directory.
  • Executing installers from a directory that only contains the installer. Do not execute installers from a download directory where other unrelated files are present. Compare the files integrity hashes with those shown on the PDH download site.
  • Installing products into directories that have limited write access to prevent unauthorized tampering prior to using the installed files.

For further information and detailed steps, see this knowledge base article

Procedure

  1. Go to https://developer.arm.com/downloads/view/FM000A.

    Check that the version shown is 11.26.
    If another version is shown, click the Revision drop-down under Releases and select the above version.

  2. Click Download Now to download the package.
  3. Choose your preferred download method and click Download.
  4. Select where to save the package to and click Save. Arm recommends that you choose an empty directory with limited write permissions.
  5. Navigate to the downloaded file, and verify its MD5 checksum against the checksum listed on PDH.
  6. Extract the downloaded file contents into the same directory.
  7. If you are installing to a directory other than the default, Arm recommends that you create the target installation directory first and set its permissions to give write access only to the user performing the installation.
  8. Execute the installer from within that directory.

For more information on installing and configuring, see the Installation chapter in the Fast Models documentation.

Notice of future changes

  • Support for the Arm v7-A architecture and CPU models implementing Arm v7-A are deprecated and will be removed in a future release. Specifically, the following CPU models are deprecated:

    • Cortex-A5
    • Cortex-A7
    • Cortex-A8
    • Cortex-A9
    • Cortex-A15
    • Cortex-A17

  • Support for the following models is deprecated and they will be removed in a future release:

    • BP135TZSwitchControl
    • BP141_TZMA
    • BP147_TZPC
    • CCI400
    • CCI500
    • CCI550
    • CCN502
    • CCN504
    • CCN508
    • CCN512
    • DP500
    • DP550
    • DP650
    • GFC100
    • ICS307
    • Mali-G51
    • Mali-G72
    • Mali-T624
    • Mali-T880
    • PL030 RTC
    • PL108 MCI
    • PL340 DMC
    • SMMU-400
    • V550

  • Support for gcc 7.3 is deprecated and will be removed in a future release.

  • Support for Red Hat Enterprise Linux 7 as a host operating system is deprecated, and will be removed in a future release.

  • Support for cache state modelling beyond CPUs is deprecated, and support will be removed in a future release.

  • The ability to disable Timing Annotation (TA) (via the use of the FASTSIM_DISABLE_TA environment variable) will be removed in a future release.

    TA is enabled by default in Fast Models. TA allows for accounting of delays added to transactions in the platform, as well as for Cycles Per Instruction (CPI) being different to 1 and supporting integration with SystemC TLM environments.

  • The --save and --restore command-line options are deprecated and will be removed in a future release.

  • The following example platforms are deprecated and will be removed in a future release:

    • SGI-575
    • SGM-775
    • SGM-776
    • RD_E1_Edge
    • RD_N1_Edge
    • RD_V1
    • TC0
    • TC1

  • The Versatile Express (VE) example platforms are deprecated and will be removed in a future release.

  • [SDDKW-91102]  In the next release, some non-alphanumeric characters that are illegal in component names will be substituted with an underscore when passed to model APIs.

  • [SDDKW-91023]  The CADI debug API and Model Debugger have been deprecated for some time, and we expect to remove support for these in Fast Models 11.28. If you are still using CADI, you will need to migrate to the new Iris API for debug access to models.

  • [SDDKW-89778]  The "dmi-container-type" parameter on LISA components has been deprecated and will be removed in 11.27. After removal, the behaviour will be as if the "dmi-container-type" was set to "FullAttr" value. "TZAttr" value for this parameter is unsafe to use in most cases except maybe very simple platforms.

  • [SDDKW-88976]  The Embedded Trace Extension (ETE) is now enabled in each core using the parameter "cpu.has_ete" and configured using "cpu.ete.<parameter_name>". The ETE plugin is now deprecated, and is expected to be removed in FM 11.27.

    The Scalable Vector Extension (SVE) is now enabled in each core using the parameter "cpu.has_sve" and configured using "cpu.sve.<parameter_name>". The SVE plugin is now deprecated, and is expected to be removed in FM 11.27.

  • [SDDKW-88281]  GICv3IRI and GICv3IRI_Filter models have been removed in this release. You can use GIC_IRI and GIC_IRI_Filter instead.

New Features and Enhancements

General changes in Fast Models 11.26

  • [SDDKW-91091]  Fast Models 11.26 includes some refactoring of the binary translation engine to improve performance and maintainability long-term. In the near-term you may notice some variation in performance over the next few releases. Please report any significant performance degradation to Arm Support via https://support.developer.arm.com/ and we will address it in a future release.

Changes to specific models or components in Fast Models 11.26

  • [SDDKW-91199]  It is now possible to export the PVMetaDataController as an EVS component, and integrate this into platforms similar to other EVS components.

  • [SDDKW-91198]  Preliminary support has been added for the Ethos-U85 model.

  • [SDDKW-91100]  Preliminary support has been added for the Mali-G78AE model.

  • [SDDKW-91099]  The pa_size parameter has been added to the Cortex-R82 model. This allows a configurable physical address size as supported by later revisions of the CPU.

  • [SDDKW-91098]  Support has been added for the following models:

    • Cortex-X925
    • Cortex-A725
    • Neoverse-N3
    • Mali-G725

  • [SDDKW-91097]  CMN700:

    • Added address compare hashed regions in HN-F SAM for power of 2 hashing across SN-Fs. A new trace 'HNSAM_Target_SNF' has been added. This trace displays the target SN-F from HN-F SAM.
    • Address masking is now supported in address compare hashed regions in HN-F SAM
    • Up to 16 hashed regions are supported in HN-F SAM

  • [SDDKW-90657]  Support has been added to MMU-S3 for:

    • DPT Architectural Feature
    • Edge Triggered Integration mode

  • [SDDKW-89704]  An issue has been fixed where PPUs in Arm v9-A cluster models would erroneously print a warning message: "PPU_coreX : tried changing opmode when opmode is disabled (NUM_OPMODE=0)"

  • [SDDKW-89458]  CMN700: Added a new parameter, a4s_logicalid, to specify the A4S Logical Device ID to assign to a CCG Node.

  • [SDDKW-89294]  CMN650, CMN650R2, CMN700: Corrected CFGM por_info_global reset values for the rsp_vc_num and dat_vc_num fields.

  • [SDDKW-89165]  The ARMCortexA510CT_CortexA710CT_CortexX2CT component incorrectly included a pmbirq port. This has been removed.

Limitations

This section contains information on model limitations for models that are new in this release or new limitations that have been identified for existing models. For more information on model limitations please refer to the Fast Models Reference Guide in this release package or available on developer.arm.com

  • [SDDKW-63219]  CMN600, CMN600AE, and CMN650 incorrectly look at the upper 32-bits of the APB address. The upper address bits are expected to be 0 and the APB address is an offset that does not include PERIPHBASE.

    CMN650R2, CI700, CMN700 and future generations incorrectly look at the upper 32-bits of the APB address. The APB address includes PERIPHBASE.

Other issues

  • [SDDKW-46555]  CT model issues:

    • On Linux, a program which dynamically loads a model shared object file using dlopen() must be linked with the pthread library ( -lpthread), even if the program is otherwise single-threaded. Failure to do so may result in a segmentation fault, or other abnormal termination, possibly with the message 'Enable multithreading to use std::thread: Operation not permitted', depending on the libstdc++ version. The underlying defect is a libstdc++ bug (#67791) which has been exposed by migrating the model to C++11 threads.
